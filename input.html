Git Commands:

git clone 'github repository link'
copy the git repository in our machine as local repository.
git add 'file name'
when we make changes in our local repository and if we want to commit these changes then first these has be in the stage area, git add will make the changes to the stage area.
git add *
these command will add all the files to whom changes were made into the stage area.
git commit -m 'commit message'
commit with meassage to all the changes present in the stage area. Without message command will fail.
git commit -am 'commit message'
commit -am will combine the above two commands (add + commit) in one command, we don't need to run the add and commit command separately.
git status
will provide the status of the commit, if there has been any modification on the file or not.
git log
will provide all the changes and commits.
git push
this command will push the changes to the master repository from the local repository for which the changes has been commited.
git pull
pull command will gather the updated copy of the master repository to the local repository.
git reset --hard 'hash code'
reset command will take us to the previous commited code to verify what changes we have done.
git reset --hard origin/master
reset --hard origin/master: will take us to the current copy of the master repository from github.
git revert 'hash code'
The git revert command is used for undoing changes to a repository's commit history. Other 'undo' commands like, git checkout and git reset, move the HEAD and branch ref pointers to a specified commit. Git revert also takes it, however, git revert does not move ref pointers to this commit. A revert operation will take the specified commit, inverse the changes from that commit, and create a new "revert commit". The ref pointers are then updated to revert commit making it the tip of the branch.
git fetch
fetch command will take the updated copy of the master to local but the changes will not reflect in our local repository until we merge the changes with the master repository.
git merge origin/master
after the fetch of the updated master copy, merge origin/master: will update the changes to the local repository. --Note: Alternative to fetch and merge commands we can use "git pull" which will take the current copy and s.
git config --global user.name "John Doe"
You can run the "git config" command with the --global flag; this will make sure all of your future commits use the given information.
git config --global user.email "john@doe.org"
You can run the "git config" command with the --global flag; this will make sure all of your future commits use the given information.
git commit --amend --reset-author
This command will allow us to edit the commit done by the author.
git branch
branch command will let us know how many branches are present.
git branch 'new branch name'
branch 'new branch name': will create a new branch.
git checkout 'branch name'
checkout command will help us to switch between the branches.
git push --set-upstream origin 'branch name'
This command will create a new sub repository in GitHub with the specified branch name (in other words it will create a upstream branch in Github for our newly created branch in order to push the changes).
