Coding For Everyone:

Git Commands:



git clone 'github repository link'

git clone copy the git repository in our machine as local repository.

git add 'file name'

When we make changes in our local repository and if we want to commit these changes then first these has be in the stage area, git add will make the changes to the stage area.

git add *

git add * these command will add all the files to whom changes were made into the stage area.

git commit -m 'commit message'

git commit with message to all the changes present in the stage area. Without message command will fail.

git commit -am 'commit message'

git commit -am will combine the above two commands (add + commit) in one command, we don't need to run the add and commit command separately.

git status

git status will provide the status of the commit, if there has been any modification on the file or not.

git log

git log will provide all the changes and commits.

git push

git push this command will push the changes to the master repository from the local repository for which the changes has been commited.

git pull

git pull command will gather the updated copy of the master repository to the local repository.

git reset --hard 'hash code'

git reset --hard command will take us to the previous commited code to verify what changes we have done.

git reset --hard origin/master

reset --hard origin/master will take us to the current copy of the master repository from github.

git revert 'hash code'

The git revert command is used for undoing changes to a repository's commit history. Other 'undo' commands like, git checkout and git reset, move the HEAD and branch ref pointers to a specified commit. Git revert also takes it, however, git revert does not move ref pointers to this commit. A revert operation will take the specified commit, inverse the changes from that commit, and create a new "revert commit". The ref pointers are then updated to revert commit making it the tip of the branch.

git fetch

git fetch command will take the updated copy of the master to local but the changes will not reflect in our local repository until we merge the changes with the master repository.

git merge origin/master

after the fetch of the updated master copy, merge origin/master will update the changes to the local repository.

Note: Alternative to fetch and merge commands we can use "git pull" which will take the current copy and merge the changes.

git config --global user.name "John Doe"

You can run the git config command with the --global flag; this will make sure all of your future commits use the given information.

git config --global user.email "john@doe.org"

You can run the git config command with the --global flag; this will make sure all of your future commits use the given information.

git commit --amend --reset-author

git commit --amend command will allow us to edit the commit done by the author.

git branch

git branch command will let us know how many branches are present.

git branch 'new branch name'

git branch 'new branch name' will create a new branch.

git checkout 'branch name'

git checkout command will help us to switch between the branches.

git push --set-upstream origin 'branch name'

git push --set-upstream command will create a new sub repository in GitHub with the specified branch name (in other words it will create a upstream branch in Github for our newly created branch in order to push the changes).

git branch -d localBranchName

git branch -d command will delete the specified branch from your local repository.

git push origin --delete remoteBranchName

git push origin --delete command will delete the specified branch from your remote repository (i.e. inside github).

git checkout -b localBranchName sha

git checkout -b command will help to restore the deleted specified branch from your local repository. When you've just deleted the branch you'll see something like this in your terminal - "Deleted branch 'your-branch' (was 'sha')". Example: Deleted branch section (was 4af91d1). Here sha is 4af91d1.



Linux Commands:



tar commands
$ tar cvf archive_name.tar dirname/

Create a new tar archive.

$ tar xvf archive_name.tar

Extract from an existing tar archive.

$ tar tvf archive_name.tar

View an existing tar archive.

Examples: tar usage

grep command
$ grep -i "the" demo_file

Search for a given string in a file (case in-sensitive search).

$ grep -A 3 -i "example" demo_text

Print the matched line, along with the 3 lines after it.

$ grep -r "ramesh" *

Search for a given string in all files recursively.

Examples: grep usage

find command
# find -iname "MyCProgram.c"

Find files using file-name ( case in-sensitve find).

$ find -iname "MyCProgram.c" -exec md5sum {} \

Execute commands on files found by the find command.

# find ~ -empty

Find all empty files in home directory.

Examples: find usage

ssh command
ssh -l jsmith remotehost.example.com

Login to remote host.

ssh -v -l jsmith remotehost.example.com

Debug ssh client.

$ ssh -V

Display ssh client version.

Examples: ssh usage

export command
$ export | grep ORACLE declare -x ORACLE_BASE="/u01/app/oracle" declare -x ORACLE_HOME="/u01/app/oracle/product/10.2.0" declare -x ORACLE_SID="med" declare -x ORACLE_TERM="xterm"

To view oracle related environment variables.

$ export ORACLE_HOME=/u01/app/oracle/product/10.2.0

To export an environment variable.

ls command
$ ls -lh

Display filesize in human readable format (e.g. KB, MB etc.,).

$ ls -ltr

Using ls -ltr, order Files Based on Last Modified Time (In Reverse Order).

$ ls -F

Using ls -F, visual Classification of Files With Special Characters.

Examples: ls usage

pwd command

pwd is Print working directory or Present working directory. What else can be said about the good old pwd who has been printing the current directory name for ages.

cd command
cd -

Use cd - to toggle between the last two directories.

shopt -s cdspell

Use shopt -s cdspell to automatically correct mistyped directory names on cd.

Examples: cd usage

To create a *.gz compressed file.

$ gzip -d test.txt.gz

To uncompress a *.gz file.

$ gzip -l *.gz

Using gzip -l, display compression ratio of the compressed file.

unzip command

$ unzip -l jasper.zip

Both ftp and secure ftp (sftp) has similar commands. To connect to a remote server and download multiple files, do the following.

ftp> mls *.html -

To view the file names located on the remote server before downloading, mls ftp command as shown below.

Examples: ftp usage

ps command
$ ps -ef | more

ps command is used to display information about the processes that are running in the system. While there are lot of arguments that could be passed to a ps command, following are some of the common ones. To view current running processes.

$ ps -efH | more

To view current running processes in a tree structure. H option stands for process hierarchy.

free command
$ free

This command is used to display the free, used, swap memory available in the system. Typical free command output. The output is displayed in bytes.

$ free -g

If you want to quickly check how many GB of RAM your system has use the -g option. -b option displays in bytes, -k in kilo bytes, -m in mega bytes.

$ free -t

If you want to see a total memory ( including the swap), use the -t switch, which will display a total line as shown below.

top command
$ top -u oracle

top command displays the top processes in the system ( by default sorted by cpu usage ). To sort top output by any column, Press O (upper-case O) , which will display all the possible columns that you can sort by as shown below. To displays only the processes that belong to a particular user use -u option. The following will show only the top processes that belongs to oracle user.

Examples: top usage

df command
$ df -k

Displays the file system disk space usage. By default df -k displays output in bytes.

$ df -h

df -h displays output in human readable form. i.e size will be displayed in GB’s.

$ df -T

Use -T option to display what type of file system.

kill command
Step 1: $ ps -ef | grep vim
Step 2: $ kill -9 7243

Use kill command to terminate a process. First get the process id using ps -ef command, then use kill -9 to kill the running Linux process as shown below. You can also use killall, pkill, xkill to terminate a unix process.

Examples: kill usage

rm command
$ rm -i filename.txt

Get confirmation before removing the file.

$ rm -i file*

It is very useful while giving shell metacharacters in the file name argument. Print the filename and get confirmation before removing the file.

$ rm -r example

Following example recursively removes all files and directories under the example directory. This also removes the example directory itself.

cp command
$ cp -p file1 file2

Copy file1 to file2 preserving the mode, ownership and timestamp.

$ cp -i file1 file2

Copy file1 to file2. if file2 exists prompt for confirmation before overwritting it.

mv command
$ mv -i file1 file2

Rename file1 to file2. if file2 exists prompt for confirmation before overwritting it.

Note: mv -f is just the opposite, which will overwrite file2 without prompting.

$ mv -v file1 file2

mv -v will print what is happening during file rename, which is useful while specifying shell metacharacters in the file name argument.

cat command
$ cat file1 file2

You can view multiple files at the same time. Following example prints the content of file1 followed by file2 to stdout.

$ cat -n /etc/logrotate.conf

While displaying the file, following cat -n command will prepend the line number to each line of the output.

chmod command
$ chmod ug+rwx file.txt

chmod command is used to change the permissions for a file or directory. Give full access to user and group (i.e read, write and execute ) on a specific file.

$ chmod g-rwx file.txt

Revoke all access for the group (i.e read, write and execute ) on a specific file.

$ chmod -R ug+rwx file.txt

Apply the file permissions recursively to all the files in the sub-directories.

Examples: chmod usage

chown command
$ chown oracle:dba dbora.sh

chown command is used to change the owner and group of a file. To change owner to oracle and group to db on a file. i.e Change both owner and group at the same time.

$ chown -R oracle:dba /home/oracle

Use -R to change the ownership recursively.

mkdir command
$ mkdir ~/temp

Following example creates a directory called temp under your home directory.

$ mkdir -p dir1/dir2/dir3/dir4/

Create nested directories using one mkdir command. If any of these directories exist already, it will not display any error. If any of these directories doesn’t exist, it will create them.

locate command
$ locate crontab

Using locate command you can quickly search for the location of a specific file (or group of files). Locate command uses the database created by updatedb. The example below shows all files in the system that contains the word crontab in it.

When a man page for a command is located under more than one section, you can view the man page for that command from a specific section as shown below.

tail command
$ tail filename.txt

Print the last 10 lines of a file by default.

$ tail -n N filename.txt

Print N number of lines from the file named filename.txt

$ tail -f log-file

Using tail -f, view the content of the file in real time. This is useful to view the log files, that keeps growing. The command can be terminated using CTRL-C.

Examples: tail usage

su command
$ su - USERNAME

Switch to a different user account using su command. Super user can switch to any other user without entering their password.

$ su - raj -c 'ls'

Execute a single command from a different account name. In the following example, john can execute the ls command as raj username. Once the command is executed, it will come back to john’s account.

$ su -s 'SHELLNAME' USERNAME

Login to a specified user account, and execute the specified shell instead of the default shell.
